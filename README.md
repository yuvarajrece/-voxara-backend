## 🎯 Overview

Jamie is an AI voice agent that calls Loan Recovery Agents on behalf of the Admin Office of a Non-Banking Financial Company (NBFC). It collects structured daily work updates and stores them in MongoDB Atlas — without any human involvement.

### What It Collects
- Employee name
- Number of door-to-door visits completed
- Summary of today's visits
- Any queries for Admin or higher authorities (flagged Yes/No)
- Tomorrow's target or goal

---

## 🏗️ Architecture

```
LiveKit Agent (Jamie)
        ↓
GPT-4.1 Mini (LLM)
        ↓
Cartesia Sonic 3 (TTS) + Deepgram Nova-3 (STT)
        ↓
FastAPI Backend (Render)
        ↓
MongoDB Atlas (Database)
```

## ✨ Features

- 🗣️ **Multilingual Support** — English, Hindi, Hinglish, Tamil
- 🤖 **Fully Automated** — No human involvement needed
- 💾 **Auto Save** — Data saved to MongoDB after every call
- 🚩 **Query Flagging** — Admin queries marked Yes/No automatically
- 📞 **Outbound Calling** — Calls employees directly via phone
- 🔒 **Secure** — API key authentication on all endpoints
- 📊 **Call Summary** — GPT-4o generates end-of-call summary

---

## 📞 Call Flow

```
Step 1 → Welcome message + availability check
Step 2 → Confirm employee name
Step 3 → Number of door-to-door visits today
Step 4 → Summary of today's visits
Step 5 → Any queries for Admin? (Yes/No)
Step 6 → Tomorrow's target or goal
Step 7 → Save to MongoDB + close call
```

## ⚙️ Setup & Installation

### Prerequisites
- Python 3.11+
- MongoDB Atlas account
- LiveKit Cloud account
- OpenAI API account
- Render account

### Step 1: Clone the Repository
```bash
git clone https://github.com/yourusername/jamiee-backend.git
cd jamiee-backend
```

### Step 2: Create Virtual Environment
```bash
python -m venv .venv
.venv\Scripts\activate  # Windows
source .venv/bin/activate  # Mac/Linux
```

### Step 3: Install Dependencies
```bash
pip install -r requirements.txt
```

### Step 4: Create `.env` File
```bash
cp .env.example .env
# Fill in all values
```

### Step 5: Run Locally
```bash
python agent.py dev
```

BACKEND_URL=https://your-backend.onrender.com

## 🔌 API Endpoints

Base URL: `https://your-backend.onrender.com`

All endpoints require header:
```
x-api-key: your_api_key
```

| Method | Endpoint | Description |
|---|---|---|
| GET | `/` | Health check |
| POST | `/api/save-update` | Save employee daily update |
| POST | `/api/missed-call` | Log missed call |
| POST | `/api/save-summary` | Save call summary |
| GET | `/api/updates` | Get all completed updates |
| GET | `/api/queries` | Get flagged admin queries |
| GET | `/api/missed-calls` | Get missed call logs |
| GET | `/docs` | FastAPI documentation |

### Sample Request: Save Update
```json
POST /api/save-update
{
  "employee_name": "Rahul Sharma",
  "visits_count": 8,
  "visit_summary": "3 customers promised payment by Friday",
  "has_query": "No",
  "employee_query": "",
  "tomorrow_goal": "10 visits in Sector 14"
}
```

### Sample Response
```json
{
  "success": true,
  "message": "Saved to MongoDB Atlas",
  "id": "65f1a2b3c4d5e6f7a8b9c0d1"
}
```

## 🚀 Deployment

### Backend (Render)
```
1. Push api.py and requirements.txt to GitHub
2. Create new Web Service on Render
3. Connect GitHub repository
4. Build command: pip install -r requirements.txt
5. Start command: uvicorn api:app --host 0.0.0.0 --port 8000
6. Add environment variables in Render dashboard
```

### Agent (LiveKit Cloud)
```bash
# Install LiveKit CLI
winget install LiveKit.LiveKitCLI

# Authenticate
lk cloud auth

# Deploy agent
lk agent create

# Update agent
lk agent update
```
